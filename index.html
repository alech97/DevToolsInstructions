How to edit webpages using DevTools
DevTools goes here!

DevTools is a feature on the Google Chrome browser that allows developers to view and diagnose every piece of their web applications. But even for non-developers, DevTools can be very useful! Using DevTools, you can edit or change any aspect of a website on your local machine. Your changes will be lost once you refresh the webpage, but that doesn’t mean learning DevTools doesn’t have its uses!

Getting Started

  1. If you aren’t currently using Chrome, then open this webpage with Chrome. Instructions on how to download Google Chrome can be found here
  2. Open DevTools by either:
    • Right-clicking on any part of the page and clicking Inspect
    • Using the top right settings button , go to Settings > More Tools > Developer Tools
    • Press Ctrl + Shift + I (Cmd + Opt + I on Mac)
  3. Dock DevTools to the right of this page if it isn't already there.

    To do this: go to the settings panel inside DevTools and click the right dock icon.
  4. Open the Elements tab at the top of DevTools. Elements is used to inspect the HTML and style of a website
  5. Adjust the size of DevTools so that it is around half the webpage width.

How to Read HTML

Before we begin, we must understand what HyperText Markup Language (HTML) is. HTML tells a web browser how to organize and structure a website. Some websites are entirely coded in HTML, but most are only partly HTML.

HTML Tags

Tags could be anything you find on a webpage, such as buttons, text, boxes, tables, or images. Tags are written like so:

<tag>Something might go here</tag>

Every webpage is structured with these tags:

<!DOCTYPE html>
<html>
 <head>
  <!-- Information about this webpage goes here, such as its title -->
 </head>
 <body>
  <!-- Nearly all content goes in here, the body of the webpage -->
 </body>
</html>

Div

One of the most common and important tags is a div. A div is used to organize and style a box of code. You will notice that div’s are used extensively to group similar objects together.

Here is a piece of text located within a div. Let's see how to change this to something else!

Edit the HTML of a Website

  1. On the Elements page, you should see a panel on the left containing HTML code.
  2. Hover over various lines in this panel. You should notice that as you hover over different tags of code, Chrome will highlight the tag’s location on the page with colors. Locate the div that we want to change.
  3. To open this div, click on the icon to show its contents.
  4. You should see a p tagline. Open it up using the icon.

    If you do not, then there is a shortcut to locating tags within the HTML code: right-click on the sentence we want to change above and click Inspect to immediately locate it in DevTools.
  5. To change the text, double-click on the sentence or right-click and click Edit Text
  6. Type in whatever you want to and press Enter. The text should change on your webpage!

Edit the Style of a Website

Still on the Elements page, you should see the Styles panel on the right of the page. This panel is used to edit the style of a tag.

  1. At the top of this panel, underneath “element.style {“, you should see “color: red”. Click on “red”.
  2. Type in “green” and press Enter. The text should now be green.
  3. A new line should appear under “color:green”. If the new line disappears, click directly underneath “color” to make a text field appear again.
  4. Type “font-weight:bold” into the new line and press Enter. The text should now be bolded!
  5. Repeat steps 3-4 with the following styles:
    • font-size:30px
    • font-family:Verdana
    • font-variant:small-caps
    • text-decoration:underline
    • direction:rtl
    The text should now be in a capitalized, larger, green, underlined, Verdana font from right-to-left.
  6. To learn more style changes you can make, head to this CSS cheat sheet.

You now know how to edit the content and style of any webpage! If you’d like to learn more about the remaining pages of Chrome DevTools, head to the Google Chrome DevTools Documentation.
